## Run 1 — 2026-04-13

### Actual Follow-ups (9)

Real issues that should be addressed in a future PR or sprint.

1. **[absence]** `src/munin/core/manifest.py:76` — Missing project field raises MuninError and aborts entire ingest — inconsistent with other validation which warns and skips
   - **Why deferred:** Error handling inconsistency — not data loss on normal path. ⚠️ Regression from BUG-001 fix (SC-004 required validation error).
   - **Evidence:** Missing path → skip. Missing globs → skip. Missing project → raise MuninError. User with 10 sources where entry 5 has no project loses sources 6-10.
   - **Suggested fix:** Make all fields skip-on-invalid consistently, or make all required fields hard-error consistently. Document the policy.

2. **[framework]** `src/munin/cli/main.py:392` — result used after except block relying on _handle_error NoReturn annotation
   - **Why deferred:** Latent UnboundLocalError — only triggers if _handle_error annotation changes. No user-visible failure today.
   - **Evidence:** _handle_error(e) is NoReturn so mypy accepts result as bound. If annotation changes, UnboundLocalError. Same pattern in import_cmd.
   - **Suggested fix:** Move result-using code inside try block, or initialize result = None before try.

3. **[framework+resources]** `src/munin/core/ingest.py:136` — Two separate connection acquisitions per chunk with TOCTOU gap
   - **Why deferred:** No user-visible failure in single-user CLI. Real race only if concurrent ingest.
   - **Evidence:** SELECT on connection #1, DELETE+upsert on connection #2.
   - **Suggested fix:** Combine in one connection block or use SELECT ... FOR UPDATE.

4. **[framework]** `src/munin/core/ingest.py:88` — Manual DELETE before upsert_thought defeats conflict-resolution logic
   - **Why deferred:** Consistency improvement, no user-visible failure.
   - **Evidence:** Code DELETEs old row then calls upsert_thought, bypassing UPDATE path.
   - **Suggested fix:** If upsert_thought has ON CONFLICT DO UPDATE, remove manual DELETE.

5. **[security]** `src/munin/core/ingest.py:85` — Glob traversal follows symlinks, can escape source root
   - **Why deferred:** Operator-scoped footgun, not remote attack vector.
   - **Evidence:** Path.glob() follows symlinks. Symlink pointing outside source.path passes is_file() guard.
   - **Suggested fix:** Add containment check: resolved.startswith(source.path.resolve()).

6. **[security]** `src/munin/core/ingest.py:92` — No file size limit before read_text
   - **Why deferred:** Defensive hardening for hypothetical edge case.
   - **Evidence:** read_text() with no size guard on glob matches.
   - **Suggested fix:** Check stat().st_size before read_text; skip above 1 MB.

7. **[resources]** `src/munin/core/embed.py:66` — New httpx.Client per embed() call, no connection reuse
   - **Why deferred:** Performance improvement, not functional bug.
   - **Evidence:** N TCP connect cycles instead of 1. embed_batch() already does it right.
   - **Suggested fix:** Use embed_batch() or pass shared httpx.Client.

8. **[absence]** `tests/unit/test_ingest.py` — No test covers non-dry-run write path
   - **Why deferred:** Test-only impact.
   - **Evidence:** All tests use dry_run=True. Fingerprint lookup, DELETE+upsert, skip logic untested.
   - **Suggested fix:** Add three mock-based tests: new chunk, unchanged chunk, changed chunk.

9. **[absence]** `src/munin/core/ingest.py:124` — dry_run increments chunks_stored (semantically ambiguous)
   - **Why deferred:** Naming issue, not functional bug. CLI labels it correctly.
   - **Evidence:** IngestResult.chunks_stored incremented in dry_run path where nothing is stored.
   - **Suggested fix:** Add chunks_would_store field or derive from len(dry_run_chunks).
